Dropdown examples
This template shows the possibilities for using the select element (dropdown box)

The options of this select are defined in the html file, so they are static.

The options of this select are read from the nlw tool. To use this functions some additional attributes are mandatory, others are optional.
The attribute data-nlw-table (mandatory) defines the table that is used (in this case the table "languages.db").
The attribute data-nlw-keys (mandatory) defines the visible text of the options. In This case we use "column:A". The text of the options in this dropdown are the values of column A.
The attribute data-nlw-values (optional) defines the value of the options. In This case we use als "column:A". So text and value are identical.
The attribute data-nlw-start-index (optional) defines the first row that is used for the text and the value. We can define this attribute because most nlw tables will have a headline row.
The attribute data-nlw-prefix (optional) fetches the value defined for the attribute and adds it to the beginning of each option value. Use case would be a flag path that will be added before every flag name.
